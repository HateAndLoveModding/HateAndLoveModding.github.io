Qwixx in Python

The Game of Qwixx is a fun dice game that my family loves to play together. One day while we were playing, I had a crazy idea to make a slight modification for extended gameplay. The first picture is the normal gameplay. I decided to make a modification where either the numbers or colors could be randomized or both could be randomized. The randomized colors does not really change gameplay but the randomized numbers does. Since very high or low numbers are hard to get, having them be more spread out makes it easier to lock the color which means that you get more points.

To code Qwixx in Python, I used the Tkinter library to create a fully interactive graphical user interface (GUI) for multiple players. I built the code with modularized principals with Player and QwixxGame classes to handle logic and GUI elements in a maintainable way allowing each player to interact with their own color-coded scorecard. Each player is initialized with their own frame, buttons representing numbers 2â€“12 (plus a "Lock" and penalty row), and an "Undo" button to revert moves. When creating the scorecards for each player, I wrote the number and color data to a file (qwixx.txt) to ensure that all players had the exact same scorecard to ensure fairness. The dice rolling mechanism simulates six dice with different colors, calculates all valid roll sums, and highlights corresponding clickable buttons on the board. I implemented roll history and undo functionality to restore previous game states when a player accidentally clicks the wrong button. The game ends by calculating scores based on how many Xs were marked in each color row and penalties taken. On GitHub you may notice to different files, one for Linux and one for Windows. This is because I wanted to add a feature where it would save a picture of the game at the end. On Linux I used gnome-screenshot in order to accomplish this. Since I use Debian with Gnome as my main operating system, I have not made any recent attempt to update the Windows file but the Linux file should work on Windows except that it will throw an error when the "End Game" button is clicked because it is trying to take a screenshot using gnome-screenshot.
